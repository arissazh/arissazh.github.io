This project uses the P5.js library to create six different sketches. The projects can be accessed here through my sketch gallery.

There are six sketches total, each building off of the previous sketch.

  • A static pattern of white daisies
  • A static, random-colored daisy pattern across the screen--will look different every time!
  • An infinite loop of random colored daisies being placed onto the page randomly
  • An interactive screen where once clicked, random colored daisies are placed
  • A single daisy that plays a note of "Twinkle Twinkle Little Star" each time it is clicked, turning the petals purple
  • Seven daisies that change petal colors when hovered over or clicked, and each play a note corresponding to C-Major

My Development Process

  • Use p5 editor to create sketches
  • Create daisy pattern
  • Repeat, alternate random colors, alternate random positions for sketches 1-3
  • Change placement "draw" to when mouse is clicked--added function
  • Use in class examples to guide sketch 5 and 6, which involve audio

I started with creating the sketches on the p5js editor site, and then transferred them into my github repo to make sure they worked properly. Along the way, I had to start over occaisionally due to the page refreshing adn sketches not saving, as well as having to redo parts of the sketches to meet assignment requirements.

Issues

After reading the assignment requirements, I realized that I had to make sure resizing the window would also make changes to my sketches. WHen I had started the sketches, I used the default canvas sizes of 400x400, so when creating my original daisy pattern, I used a for loop that iterated five times, since I knew that five flowers would span the whole canvas size. However, after changing the canvas size to be windowWidth by windowHeight, my pattern started offsetting, probably due me using translate and mathematical calculations to move the starting position back each time.

I tried to fix the alignment of the pattern by adding values to the translate function, and while I was able to get the pattern to align, it would have messed up when resizing the window again.

I resolved this issue by adding a function that created a list of x coordinates and y coordinates where I wanted the flowers to be placed. When the flowers were drawn, they would be passed coordinates based on where in the for loop for reading through the coordinate list. This required a lot of debugging as I had to make sure the two lists were being read properly, so I used console logging quite a lot during this phase. However, once I got it working, it resolved the window resizing pattern issue!
